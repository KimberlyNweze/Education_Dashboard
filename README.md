# Education_Dashboard
# LLM-Adaptive Visualisation Selection Dashboard

A Streamlit dashboard that uses a Large Language Model (Groq API) to adaptively select and annotate visualisations of collaborative learning patterns, designed to support teacher understanding of regulatory dynamics in small-group discussions.

---

## Important — all files must be in the same folder

Every file in this repository must be placed in the same directory before running the dashboard. The code uses relative paths and will not work if files are in separate folders.

---

## Requirements

- Python 3.9 or higher
- A Groq API key (free to obtain — see below)

Install all dependencies by running:

```bash
pip install streamlit groq python-dotenv networkx pandas matplotlib numpy
```

---

## Groq API Key Setup

**The dashboard will not run without a Groq API key.** The LLM visualisation selection feature calls the Groq API to generate visualisation choices and annotations.

**Step 1 — Get a free API key**

Go to [Groq_API](https://console.groq.com), create a free account, and generate an API key.

**Step 2 — Create a .env file**

In the same folder as all your project files, create a file called `.env` (no filename, just the extension) and add the following line:

```
GROQ_API_KEY=your_actual_key_here
```

Replace `your_actual_key_here` with the key you copied from the Groq console.

**Do not share this file or upload it to GitHub** — it contains your private API key.

---

## How to Run the Dashboard

Once all files are in the same folder and your `.env` file is set up, open a terminal in that folder and run:

```bash
streamlit run Dashboard.py
```

The dashboard will open automatically in your browser. If it does not open, navigate to `http://localhost:8501` manually.

## File Descriptions

| File | Description |
|------|-------------|
| `Dashboard.py` | The main Streamlit dashboard application. Run this file to launch the dashboard. Contains all three pages: Session Overview, Session Summary, and Episode Detail. |
| `visualisations.py` | Contains all five visualisation functions (Timeline, Participation Chart, Network Graph, Stacked Bar, Heatmap). Required by the dashboard — must be in the same folder. |
| `episode_features.csv` | Pre-computed features for all 882 episodes across 28 sessions. Generated by the Feature Extraction notebook. Required by the dashboard to load episode data. |
| `Data_Exploration.ipynb` | Jupyter notebook covering initial dataset exploration — basic statistics, episode distributions, and session-level summaries. |
| `Feature_Extraction.ipynb` | Jupyter notebook that extracts temporal, participation, challenge, and regulation features from the raw dataset and produces `episode_features.csv`. |
| `baseline_comparison.ipynb` | Jupyter notebook for evaluating LLM visualisation selection against a rule-based selection baseline. |
| `evaluation_episodes.json` | The 20 stratified episodes selected for evaluation, stored as JSON. Used by the baseline comparison notebook. |
| `evaluation_episodes.csv` | The same 20 evaluation episodes in CSV format, including feature values and selection rationale. |
| `llm_log.json` | Auto-generated log of all LLM calls made through the dashboard. Each entry records the session, episode, selected visualisation, reason, teacher explanation, and timestamp. |
| `llm_test_results.json` | LLM outputs from test runs during development. Used for reference and reproducibility. |
| `test_episodes.json` | Episodes used during development testing of the LLM pipeline. |

## Dashboard Pages

**Session Overview** — high-level statistics for a selected session including episode count, total utterances, duration, regulation and challenge rates by episode, and participation breakdown.

**Session Summary** — shows all episodes in the session as colour-coded chips based on LLM results. Episodes where the LLM has been run are clickable and navigate directly to the Episode Detail page. Includes flagged patterns and the full pedagogical framework reference.

**Episode Detail** — the main analysis page. Select a session and episode, click ▶ Run LLM to generate a visualisation selection and teacher annotation, and optionally click 💬 Why did you choose this visualisation? for a detailed explanation. Supports side-by-side comparison of any two visualisation types.

## Notes
- LLM results are cached within a session and logged automatically to `llm_log.json`
- Sessions 2, 3, 7, 19, 24, 26, and 28 do not have TA (Task Analysis) annotations — the dashboard flags these automatically
- The Export CSV button in Session Summary downloads all LLM results for the current session
